# Задание №3: Контрольные вопросы

1.  Разница между модульным, интеграционным и функциональным тестированием.
*   **Модульное (Unit Testing)**: Тестирование отдельных, наименьших частей программы (функций, методов, классов) в изоляции. Цель — убедиться, что каждый компонент работает правильно.
*   **Интеграционное (Integration Testing)**: Проверка взаимодействия между различными модулями или компонентами системы. Цель — выявить проблемы, возникающие при их совместной работе.
*   **Функциональное (Functional Testing)**: Проверка функциональности программы с точки зрения пользователя, без учета внутренней структуры.
___
2.  Преимущества использования тестов в процессе разработки:
* Выявление ошибок и дифектов на этапе разработки.
* Повышение качества кода, более чистого, модульного и поддерживаемого кода.
* Уверенность, что изменения не ломают старый код.
* Документация поведения системы.
___
3.  Отличия `pytest` от `unittest`:
* `pytest` не требует наследования от `TestCase` и использования специальных `assert` методов
*  Автоматически находит тестовые файлы (начинающиеся с test_ или заканчивающиеся на _test.py) и тестовые функции/методы.
* `pytest` предоставляет информативные отчеты о прохождении тестов
___
4. **`Fixtures`** Мощный механизм для подготовки тестового окружения и очистки ресурсов. Создают данные, подключают БД, открывают файлы, настраивают mock‑объекты.
___
5. TDD, разработка через тестирование, — это методология разработки программного обеспечения, при которой тесты пишутся до написания рабочего кода
  ##### 1. Red — пишем тест, который падает.
  ##### 2. Green — пишем минимальный код, чтобы тест прошёл.
  ##### 3. Refactor — улучшаем код, сохраняя прохождение тестов.
___
6. Рефакторинг — улучшение структуры кода без изменения поведения. Можно переименовывать переменные, изменять структуру функций, удалять дублирование, улучшать читаемость и оптимизировать производительность, не опасаясь сломать что-либо.
___
7. Mock — это поддельный объект, который заменяет реальный (например, requests.get).
* можно проверять, как функция реагирует на ошибки
* можно контролировать ответы внешних сервисов
___
8. Какие проблемы могут возникнуть при тестировании функций, зависящих от внешних ресурсов, без использования Mock-объектов:
* тесты зависят от сети  нестабильные
* внешние ресурсы могут быть недоступны из-за чего тесты падают
* тесты становятся медленными
* невозможно проверить ошибки (например, 500, timeout)
___
9. Как TDD способствует улучшению дизайна кода:
* заставляет писать маленькие, чистые функции
* код становится модульным и легко тестируемым
* меньше связности между компонентами
* архитектура развивается естественно, а не хаотично
* разработчик думает о поведении кода заранее
